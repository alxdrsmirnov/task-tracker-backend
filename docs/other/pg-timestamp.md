# Типы времени в PostgreSQL и поля вроде `joined_at`

## `timestamp` без часового пояса

`timestamp without time zone` (в UI часто подписан как `timestamp`) — это набор даты и времени **без привязки к UTC/IANA**: одни и те же цифры в разных поясах — **разные моменты в мире**, если не зафиксировано соглашение «всегда пишем UTC».

## `timestamptz`

`timestamp with time zone` (`timestamptz`) — хранится **один момент на линии времени**. При отображении в текстовом протоколе PostgreSQL может участвовать пояс сессии; в приложении (драйверы, Prisma, JSON API) момент обычно передаётся как абсолютное время, а формат для клиента задаётся на стороне бэкенда.

## Точность `(n)` в `timestamptz(n)`

Число `n` — **количество десятичных знаков в доле секунды**:

- `(3)` — до **миллисекунды**;
- `(6)` — до **микросекунды**.

Фраза «6 миллисекунд» часто путается с этим: **`(6)` — это не «6 мс», а микросекундная точность**.

# Решение `@db.Timestamptz(6)` во всех моделях (Prisma)

## Плюсы

- единообразие;
- явный тип с часовым поясом в PostgreSQL;
- одна договорённость на весь проект;
- высокая точность для логов и порядка событий в одной секунде.

## Нюансы

- для типичного CRUD **микросекунды часто избыточны** — достаточно `(3)`, если нет требований к субмиллисекундному порядку или аналитике;

## Итог

Направление **верное** (`timestamptz` везде для «моментов»); выбор `(6)` vs `(3)` — про **нужную точность и размер**, а не про «правильно/неправильно».

---

## Хранение

`timestamptz` + момент события **в смысле одной точки на оси времени** (условно «UTC» при хранении внутри сервера).

## API

В JSON обычно отдают **ISO 8601 в UTC** с суффиксом **`Z`**, чтобы фронт однозначно парсил и показывал локаль через `Intl` или библиотеку. Так убирается двусмысленность «это моё локальное или серверное».

## Профиль пользователя

При необходимости напоминаний, дедлайнов «до конца дня по календарю пользователя», отчётов — хранят **`timeZone` в виде IANA** (`Europe/Moscow`), а не смещение «+3» (IANA учитывает DST). Конвертацию из UTC в эту зону делают библиотеки на сервере там, где это нужно (письма, PDF, крон); простой UI часто **сам** переводит из UTC.

## Доверие

Заголовок «текущая зона с браузера» удобен для UI; для **критичной логики** лучше опираться на **сохранённую настройку** и валидацию.
